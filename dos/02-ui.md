# Estructura visual (UI/UX)

## Principios de diseño
- Mobile-first, bloques con radios grandes, sombras suaves y bordes sutiles.
- Jerarquía clara: títulos fuertes, subtítulos en `text-slate-500`, acciones primarias en morado.
- Interacciones: hover/focus con `ring` y transiciones suaves.

## Paleta de colores (Tailwind)
- Fondo app: `bg-purple-50`
- Superficies: `bg-white`
- Primario: `bg-purple-700` (hover `bg-purple-500`)
- Acento: `bg-pink-600` (hover `bg-pink-500`)
- Bordes/rings: `ring-purple-200/50`, `border-purple-200/60`
- Estados:
  - Aprobado: tonos morados (`bg-purple-50`, `text-purple-700`)
  - Pendiente: tonos rosados (`bg-pink-50`, `text-pink-600`)
  - Rechazado: tonos rosados (`bg-pink-50`, `text-pink-600`)

## Tipografía
- Por defecto: stack del navegador (Tailwind base). 
- Uso:
  - Titulares: `text-2xl font-semibold tracking-tight`
  - Secciones/cards: `text-sm font-bold` o `font-extrabold`
  - Labels: `text-xs font-semibold text-slate-500`

## Pantallas y funcionalidades

### Login / Registro (`/login`)
- Modo login:
  - Usuario: teléfono (10 dígitos).
  - Password con toggle mostrar/ocultar.
- Modo registro (`?mode=register`):
  - Nombre completo, teléfono, correo, contraseña, confirmar contraseña, género.
  - Toggle mostrar/ocultar en password y confirmación.
  - Navegación entre formularios con enlaces.

### Dashboard (redirige) (`/dashboard`)
- Detecta perfil y rol para enviar a:
  - Admin: `/admin`
  - Socio: `/socio`

### Admin Layout (`/admin/*`)
- Drawer lateral con navegación:
  - Dashboard
  - Gestionar Socios
  - Gestionar Abonos
  - Gestionar Préstamos
  - Gestionar Actividades
  - Editar Perfil

### Admin Dashboard (`/admin`)
- Métricas reales:
  - Socios activos.
  - Total ahorrado (abonos aprobados).
  - Préstamos activos (aprobados).
- Gráfica línea “Ahorro por mes” (periodo Dic→Nov).
- Donut “Distribución” del mes actual (conteos).

### Gestionar Socios (`/admin/socios`)
- CRUD de socios con tarjetas:
  - crear socio (signup aislado para no romper sesión).
  - editar datos en `profiles`.
  - activar/desactivar.
  - búsqueda y paginación.

### Gestionar Abonos (`/admin/abonos`)
- Selector de persona:
  - incluye “Yo (Admin)” para auto-reportarse abonos.
  - incluye socios activos.
- Grilla Dic→Nov con Q1/Q2:
  - registrar/editar (admin).
  - aprobar/rechazar (admin) por cuota.
  - historial por celda.
  - fecha exacta del aporte por cuota.
  - aporte masivo (rango de meses y quincena).
  - aprobación masiva (pendientes) por rango/quincena.

### Gestionar Préstamos (`/admin/prestamos`)
- Reglas del préstamo:
  - % interés.
  - tope máximo (% de lo ahorrado).
- Solicitudes:
  - filtrar por estado.
  - aprobar/rechazar con modal de confirmación.
  - motivo de rechazo visible para el socio.

### Editar Perfil (`/admin/perfil`)
- Actualizar nombre.
- DangerZone (solo desarrollo): botón para limpieza de datos dejando solo admin.

### Socio Layout (`/socio/*`)
- Menú:
  - Abonos (pantalla principal)
  - Solicitud de préstamos

### Socio Abonos (`/socio/abonos`)
- Misma grilla Dic→Nov Q1/Q2:
  - socio envía solicitud (pendiente).
  - bloqueado si estado es pendiente/aprobado, solo reenvía cuando rechazado.
  - historial visible.
  - aporte masivo (crea solicitudes pendientes) por rango/quincena.

### Socio Préstamos (`/socio/prestamos`)
- Calcula tope máximo con:
  - total ahorrado aprobado.
  - `loan_settings.max_loan_percent`.
- Solicitud guarda el interés snapshot (`interest_rate_percent`) al crear.
- Lista de solicitudes con estado y motivo de rechazo (si aplica).
